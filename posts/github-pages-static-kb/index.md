## 起点

个人站点最重要的不是功能复杂，而是能长期稳定地承载内容。这个站点选择 GitHub Pages，是因为它适合公开内容、静态页面和低维护成本的个人主页。

## 页面分层

首页只承担索引职责：展示文章、主题笔记和公开资料。真正的正文放在独立页面中，这样读者打开一篇内容时，不会被首页上的搜索、标签和其他列表干扰。

当前结构按三类内容组织：

- 文章：记录完整主题、项目复盘和工程实践
- 笔记：沉淀持续更新的主题索引
- 资料：提供公开文件和说明入口

## 内容索引

首页通过 `data/content.json` 读取公开索引。索引只保存标题、摘要、标签、分类和目标地址，不直接承载完整正文。

这样做的好处是：

1. 首页保持轻量。
2. 搜索范围清晰。
3. 正文页面可以按博客方式独立排版。
4. 后续迁移到 Jekyll 集合或自动生成索引时成本更低。

## 自动部署

GitHub Pages 的构建流程由仓库里的工作流负责。内容提交后，构建流程会把 Jekyll 页面输出为静态站点，并由 GitHub Pages 托管。

## 长期维护原则

这个站点会优先保持简单：

- 首页负责发现内容
- 文章页负责阅读体验
- 数据索引负责搜索和导航
- 静态资源只放可公开分享的小文件

复杂功能只有在确实提升长期维护效率时才加入。
